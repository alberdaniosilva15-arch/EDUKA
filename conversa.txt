1. VISÃO GERAL DO PROJECTO
--------------------------------------------------------------------------------
Plataforma educacional com IA para estudantes angolanos.
Stack: Next.js 15 + React 19 RC + Supabase + Gemini/Groq
Ferramentas: Gerador de Trabalhos, Melhorador de Texto, Explicação Simples,
             Plano de Estudo, Gerador de Slides
Exportação: DOCX, PDF, TXT, Markdown, PPTX

2. BUGS ENCONTRADOS
--------------------------------------------------------------------------------

=== CRÍTICOS ===

[BUG-01] learning-system.js:1-9
  Severidade: CRÍTICO
  Problema: Cria cliente Supabase próprio com createClient() em vez de usar
            o admin client existente em supabase/admin.js. Resulta em duplicação
            de clientes e potenciais memory leaks em serverless.
  Correção: Reutilizar createAdminClient() do supabase/admin.js

[BUG-02] ai-provider.js (todas as funções)
  Severidade: CRÍTICO
  Problema: Sem timeout em chamadas fetch. Se a API Gemini ou Groq travar,
            o request fica pendente para sempre, bloqueando o utilizador.
  Correção: Adicionar AbortController com timeout de 30 segundos

[BUG-03] utils.js:121-152 (exportToPdf)
  Severidade: CRÍTICO
  Problema: Usa window.open() + setTimeout(250ms) para impressão. Frágil:
            - Popup blockers podem bloquear
            - 250ms pode não ser suficiente para renderizar
            - Utilizador vê janela de impressão indesejada
            - Não funciona em mobile
  Correção: Substituir por pdf-lib para geração PDF directa no browser

=== ALTOS ===

[BUG-04] utils.js:35-66 (markdownToHtml)
  Severidade: ALTO
  Problema: Parser markdown manual incompleto. Não suporta:
            - Listas numeradas (1. 2. 3.)
            - Code blocks (``` ... ```)
            - Imagens (![alt](url))
            - Links com atributos
            - Blockquotes (> texto)
            - Tabelas
  Correção: Usar biblioteca 'marked' para parsing robusto

[BUG-05] utils.js:157-218 (exportToDocx)
  Severidade: ALTO
  Problema: Geração DOCX extremamente básica:
            - Sem formatação de página (A4, margens)
            - Sem página de capa
            - Sem cabeçalhos/rodapés
            - Sem listas numeradas
            - Sem blockquotes ou code blocks
            - Sem imagens ou tabelas
  Correção: Expandir com formatação profissional completa

[BUG-06] utils.js:107-116 (exportToTxt)
  Severidade: ALTO
  Problema: Regex .replace(/_ /g, '') não remove marcadores markdown
            correctamente. Itálicos *texto* não são removidos.
  Correção: Corrigir regex para remover todos os marcadores

[BUG-07] api/slides/route.js:26-38
  Severidade: ALTO
  Problema: Prompt construído inline em vez de usar buildXxxPrompt()
            como os outros routes. Inconsistente e difícil de manter.
  Correção: Criar buildSlidesPrompt() em prompts.js

[BUG-08] api/estudo/route.js:27-37
  Severidade: ALTO
  Problema: Mesmo problema que BUG-07 — prompt inline.
  Correção: Criar buildStudyPrompt() em prompts.js

=== MÉDIOS ===

4. BIBLIOTECAS OPEN SOURCE RECOMENDADAS
--------------------------------------------------------------------------------

| Biblioteca | Tamanho | Uso                          | Justificação                           |
|-----------|---------|------------------------------|---------------------------------------|
| marked    | ~30KB   | Parser markdown              | Substitui 30+ linhas de regex frágeis |
| pdf-lib   | ~500KB  | Geração PDF                  | Substitui window.print, fiável        |

Porquê estas:
- marked: parser markdown mais leve e rápido do ecossistema JS
- pdf-lib: geração PDF 100% client-side, sem dependências externas

O que NÃO adicionar:
- puppeteer/playwright: pesados, desnecessários
- html2pdf.js: depende de html2canvas, pesado
- jspdf: mais pesado que pdf-lib
